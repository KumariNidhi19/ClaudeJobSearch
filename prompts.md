# Prompts

These are the instructions Cowork uses. You paste `SCHEDULED_TASK` once when
creating the daily task (see `README.md`). `PROFILE`, `RESUME`, and
`JOB PREFERENCES` live in their own files in this repo/project — Cowork reads
them at run time, so you don't paste them here.

---

## SCHEDULED_TASK

Read `profile.md`, `resume.md`, and `job-preferences.md` in this project (or
GitHub repo). Then search the web for **currently open** Product Designer /
UI-UX / UX / AI Product Designer roles that fit.

**Locations:** open to ALL locations — Pune preferred, but do not exclude any
city or country. Include on-site (any location), hybrid, and remote roles.

**Where to search — cover all of these each run:**
- Job boards: **LinkedIn Jobs, Naukri, Indeed India, Wellfound (AngelList),
  Instahyre, Cutshort, Hirist, Foundit (Monster), Glassdoor, Y Combinator
  "Work at a Startup".**
- **Career pages of prominent companies that hire product designers**, biased to
  my domains (AI/GenAI, enterprise SaaS, pharma/health, B2B/B2C SaaS). Check a
  rotating set such as: Google, Microsoft, Adobe, Atlassian, Salesforce, Intuit,
  Uber, Amazon, PayPal, Booking.com, Canva, Figma, Notion, Postman, ServiceNow,
  Freshworks, Zoho, Razorpay, CRED, PhonePe, Groww, Flipkart, Swiggy, Zomato,
  Meesho, Myntra, Sprinklr, BrowserStack, Chargebee, ThoughtSpot, Innovaccer,
  Fractal, Zeta, Navi, and other reputable product companies. You may include
  strong companies not on this list too.

Focus on roles posted or updated in the **last 14–21 days**.

**Apply the work-schedule rules in `job-preferences.md` strictly:**
- Rank 2–3 day hybrid roles highest (Tier 1), then standard 5-day-week roles
  (Tier 2).
- Reject 6-day / alternate-Saturday roles UNLESS the employer is BCG, JPMorgan
  Chase, PwC, Deloitte, Capgemini, or a comparable global-tier firm (Tier 3).
- If the schedule isn't stated, include the role but set schedule to "unknown"
  and add a short note in the Flag column to verify before applying.

Before adding anything, **read the existing `application-tracker.md`** (all
three tables) and skip any posting already listed (same URL, or same company +
title). Only append genuinely new roles.

**Target 20 new roles per run.** If 20 genuinely qualifying, currently-open roles
don't exist today, add as many real ones as you find — never invent postings,
pad with off-target roles, or reuse dead links to reach 20.

For each new role, append one row to the **Inbox** table in
`application-tracker.md`, newest first (best fit first), with: date found
(today), tier, title, **company as a markdown link to its careers page or site**,
location, work mode, schedule, fit score (0–100), a short flag if anything needs
checking, the **direct job-posting link** in the Link column, and status = "new".

Only include roles with a real, working URL you actually found via search. When
done, save the file and give me a 3-line summary: how many new roles added, the
single best match, and any roles you flagged as "unknown" schedule.

## COVER_NOTE (on-demand helper — not scheduled)

Write a 90–120 word, warm-but-professional application note for the role I name,
using concrete wins from `resume.md` (the PLS tool with 91% SME approval, the
190% active-user lift, or the voice-first booth assistant — whichever fits
best). No clichés, no "I am writing to apply."
